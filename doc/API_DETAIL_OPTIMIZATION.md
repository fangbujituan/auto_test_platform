# 接口详情页面优化说明

## 优化内容

### 新布局结构

接口详情页面已按照以下结构重新设计：

#### 第一行：请求配置行
- **请求方式下拉框**：支持 GET、POST、PUT、DELETE、PATCH
- **URL输入框**：显示和编辑完整的请求URL
- **发送按钮**：执行接口测试请求
- **保存按钮**：保存接口配置的修改

#### 第二行：接口描述
- 多行文本输入框，用于编辑接口描述信息

#### 第三行：请求参数（多页签）
- **Query Params**：查询参数配置（JSON编辑器）
- **Headers**：请求头配置（JSON编辑器）
- **Body**：请求体配置（JSON编辑器）
  - 支持选择 body_type：JSON、Form、Raw
- **前置脚本**：待开发功能
- **后置脚本**：待开发功能

#### 第四行：响应结果
- **响应头部**：
  - 左侧显示"响应结果"标题
  - 右侧显示状态码标签（默认200）
- **响应内容（多页签）**：
  - **Body**：响应体内容（JSON格式化显示）
  - **Headers**：响应头信息
  - **Info**：响应详细信息（状态码、耗时、响应大小、时间戳）

## 主要功能

### 1. 实时编辑
- 所有请求参数都可以直接在页面上编辑
- 修改后点击"保存"按钮持久化到数据库

### 2. 快速测试
- 点击"发送"按钮立即执行接口测试
- 测试结果实时显示在响应区域
- 显示状态码、响应时间、响应大小等信息

### 3. 响应式布局
- 请求参数和响应结果区域采用弹性布局
- 自动适应不同屏幕尺寸
- 支持滚动查看长内容

## 技术实现

### 响应式数据
```javascript
// 可编辑的接口数据
const editableApi = ref({
  method: 'GET',
  fullUrl: '',
  description: '',
  headers: {},
  params: {},
  body: {},
  body_type: 'json'
})

// 响应数据
const responseData = ref({
  statusCode: 200,
  body: null,
  headers: null,
  info: null,
  duration: null,
  size: null,
  timestamp: null
})
```

### 核心方法

1. **sendRequest()**：发送测试请求
   - 解析完整URL
   - 调用后端测试接口
   - 更新响应数据
   - 显示请求耗时

2. **saveApiChanges()**：保存接口修改
   - 解析URL为 base_url 和 path
   - 更新接口配置
   - 刷新目录树

3. **watch(currentApi)**：监听接口切换
   - 自动加载接口数据到编辑区
   - 清空之前的响应数据

- 所有区域支持独立滚动

- 清晰的视觉层次

## 使用说明

1. 在左侧目录树中选择一个接口
2. 右侧自动显示接口详情
3. 可以直接修改请求方式、URL、描述等信息
4. 在请求参数页签中配置 Query Params、Headers、Body
5. 点击"发送"按钮测试接口
6. 查看响应结果的 Body、Headers、Info
7. 修改完成后点击"保存"按钮保存更改
