# Bug管理前端使用指南

## 概述

Bug管理前端页面已完成开发，提供了完整的Bug管理功能界面。

## 已完成的工作

### 1. API接口层 ✅
**文件**: `client/src/api/bug.js`

实现了所有Bug管理相关的API调用：
- `getBugs()` - 获取Bug列表
- `getBug()` - 获取Bug详情
- `createBug()` - 创建Bug
- `updateBug()` - 更新Bug
- `deleteBug()` - 删除Bug
- `resolveBug()` - 解决Bug
- `reopenBug()` - 重新打开Bug
- `getBugStatistics()` - 获取统计信息

## 页面功能

### 1. 统计概览
页面顶部显示5个统计卡片：
- **总计** - 显示所有Bug数量
- **待处理** - 显示待处理Bug数量（橙色）
- **处理中** - 显示处理中Bug数量（蓝色）
- **已解决** - 显示已解决Bug数量（绿色）
- **严重** - 显示严重级别Bug数量（红色）

### 2. 筛选和搜索
提供多维度筛选：
- **状态筛选** - 5种状态可选
- **优先级筛选** - 4个级别可选
- **严重程度筛选** - 5个级别可选
- **关键词搜索** - 搜索标题、描述、模块
- **重置按钮** - 清空所有筛选条件

### 3. Bug列表
表格展示所有Bug：
- 支持点击行查看详情
- 状态、优先级、严重程度使用不同颜色的标签显示
- 每行提供查看、编辑、删除操作按钮

### 4. Bug详情
点击Bug后显示详情对话框：
- 完整的Bug信息
- 格式化的复现步骤
- 解决信息（如果已解决）
- 可以直接在详情页进行操作：
  - 解决Bug
  - 重新打开Bug
  - 编辑Bug

### 5. 创建Bug
点击"新建Bug"按钮：
- 填写Bug基本信息
- 选择状态、优先级、严重程度
- 填写分类、模块、环境、版本
- 详细描述复现步骤、预期结果、实际结果
- 表单验证确保必填项

### 6. 编辑Bug
点击编辑按钮：
- 加载现有Bug信息
- 修改任意字段
- 保存更新

### 7. 解决Bug
在详情页点击"解决Bug"：
- 选择解决方案
- 填写解决说明
- 自动更新状态为"已解决"
- 记录解决时间

### 8. 重新打开Bug
对于已解决的Bug：
- 点击"重新打开"按钮
- 确认后状态变为"重新打开"
- 清除解决信息

### 9. 删除Bug
点击删除按钮：
- 弹出确认对话框
- 确认后删除Bug
- 刷新列表和统计

## 访问路径

### 从项目详情页进入


## 使用流程

### 创建Bug流程
1. 点击"新建Bug"按钮
2. 填写Bug标题（必填）
3. 选择优先级和严重程度（必填）
4. 填写详细描述和复现步骤
5. 点击"确定"创建

### 处理Bug流程
1. 在列表中找到Bug
2. 点击查看详情
3. 点击"解决Bug"
4. 选择解决方案
5. 填写解决说明
6. 确认解决

### 查询Bug流程
1. 使用顶部筛选器
2. 选择状态、优先级、严重程度
3. 或输入关键词搜索
4. 点击"搜索"按钮
5. 查看筛选结果

## 技术实现

### 组件技术栈
- **Vue 3** - Composition API
- **Element Plus** - UI组件库
- **Vue Router** - 路由管理
- **Axios** - HTTP请求

### 状态管理
使用Vue 3的ref和computed进行状态管理：
- `bugList` - Bug列表数据
- `statistics` - 统计数据
- `currentBug` - 当前选中的Bug
- `filterForm` - 筛选条件
- `form` - 表单数据

### 数据流
1. 页面加载时获取Bug列表和统计信息
2. 筛选时重新请求数据
3. 创建/编辑/删除后刷新列表和统计
4. 所有操作都有loading状态和错误处理

## 响应式设计

- 统计卡片使用grid布局，自动适应屏幕宽度
- 表格支持横向滚动
- 对话框在小屏幕上自动调整宽度
- 筛选表单在小屏幕上自动换行
