# Bug管理新界面使用指南

## 🎨 界面概览

新的Bug管理界面采用三栏布局设计，与接口管理保持一致的用户体验。

### 布局结构

```
┌─────────────┬──────────────────┬────────────────────────┐
│             │                  │                        │
│  功能模块   │   目录树 + Bug   │     Bug详情展示        │
│   导航栏    │     列表区域     │                        │
│             │                  │                        │
└─────────────┴──────────────────┴────────────────────────┘
   200px           350px              自适应宽度
```

## 📋 功能特性

### 1. 左侧功能模块栏 (200px)

**项目信息**
- 项目图标和名称展示
- 快速识别当前项目

**模块导航菜单**
- 接口管理
- Bug管理 (当前)
- 自动化测试 (开发中)
- 测试用例
- 环境配置 (开发中)

**特点**
- 模块间快速切换
- 当前模块高亮显示
- 禁用模块灰色显示

### 2. 中间目录树区域 (350px)

**顶部工具栏**
- 搜索框：快速搜索Bug
- 新建按钮：创建目录或Bug
- 更多操作：刷新、初始化目录

**目录树展示**
- 📁 目录节点
  - 折叠/展开图标
  - 目录名称
  - 统计信息 (子目录数、Bug数)
  - 操作菜单
    - 新建子目录
    - 新建Bug
    - 重命名
    - 删除

- 🐛 Bug节点
  - 优先级标签 (P0/P1/P2/P3)
  - Bug标题
  - 状态标签 (待处理/处理中/已解决等)
  - 操作菜单
    - 编辑
    - 解决
    - 重新打开
    - 删除

**优先级标签颜色**
- P0 (紧急): 红色 danger
- P1 (高): 橙色 warning
- P2 (中): 默认色
- P3 (低): 灰色 info

**状态标签颜色**
- 待处理: 橙色 warning
- 处理中: 蓝色 primary
- 已解决: 绿色 success
- 已关闭: 灰色 info
- 重新打开: 红色 danger

### 3. 右侧详情面板 (自适应)

**空状态**
- 未选择Bug时显示提示信息
- 引导用户选择Bug

**Bug详情展示**

**顶部标题区**
- 渐变色背景 (紫色渐变)
- 三个大标签：优先级、严重程度、状态
- Bug标题
- 操作按钮：解决、重新打开、编辑、删除

**详情内容区 (可滚动)**

1. **基本信息**
   - Bug ID
   - 状态、优先级、严重程度
   - 分类、模块
   - 测试环境、发现版本
   - 创建时间

2. **Bug描述**
   - 灰色背景卡片展示
   - 支持多行文本

3. **复现步骤**
   - 灰色背景卡片
   - 保留格式的文本展示

4. **预期与实际结果**
   - 左右分栏展示
   - 预期结果：绿色左边框
   - 实际结果：红色左边框

5. **解决信息** (已解决的Bug)
   - 解决方案标签
   - 解决时间
   - 解决说明

## 🎯 交互流程

### 创建Bug流程

1. 点击中间栏顶部的"+"按钮
2. 选择"新建Bug"
3. 填写Bug表单
   - Bug标题 (必填)
   - Bug描述
   - 状态、优先级、严重程度 (必填)
   - 分类、模块
   - 测试环境、发现版本
   - 复现步骤
   - 预期结果、实际结果
4. 点击"确定"提交

### 查看Bug详情

1. 在中间目录树中点击Bug节点
2. 右侧面板自动加载并显示Bug详情
3. 可以直接在详情页进行操作

### 编辑Bug

**方式一：通过节点菜单**
1. 点击Bug节点的"..."按钮
2. 选择"编辑"
3. 修改表单内容
4. 点击"确定"保存

**方式二：通过详情页**
1. 选中Bug查看详情
2. 点击顶部"编辑"按钮
3. 修改表单内容
4. 点击"确定"保存

### 解决Bug

**方式一：通过节点菜单**
1. 点击Bug节点的"..."按钮
2. 选择"解决"
3. 选择解决方案
4. 填写解决说明
5. 点击"确定"

**方式二：通过详情页**
1. 选中Bug查看详情
2. 点击顶部"解决"按钮
3. 选择解决方案
4. 填写解决说明
5. 点击"确定"

### 目录管理

**创建目录**
1. 点击"+"按钮选择"新建目录"
2. 输入目录名称和描述
3. 点击"确定"

**创建子目录**
1. 点击目录节点的"..."按钮
2. 选择"新建子目录"
3. 输入目录名称和描述
4. 点击"确定"

**在目录下创建Bug**
1. 点击目录节点的"..."按钮
2. 选择"新建Bug"
3. Bug会自动关联到该目录

- 详情页顶部使用紫色渐变背景

**当前选中**
- 选中的Bug节点高亮显示
- 蓝色背景标识当前项

**响应式布局**
- 小屏幕自动调整中间栏宽度
- 保证内容可读性

### 3. 用户体验

**空状态提示**
- 未选择Bug时显示友好提示
- 引导用户操作

**加载状态**
- 数据加载时显示loading动画

**操作反馈**
- 成功/失败消息提示

## 🔧 技术实现

### 组件结构

```vue
<template>
  <main-layout>
    <div class="bug-management-layout">
      <!-- 左侧模块栏 -->
      <div class="left-sidebar">...</div>
      
      <!-- 中间目录树 -->
      <div class="middle-panel">...</div>
      
      <!-- 右侧详情 -->
      <div class="right-panel">...</div>
    </div>
  </main-layout>
</template>
```

### 数据流

1. **加载目录树**
   - 调用 `getFolderTree(projectId)` API
   - 递归组装目录和Bug数据
   - 渲染到 el-tree 组件

2. **选择Bug**
   - 监听树节点点击事件
   - 调用 `getBug(projectId, bugId)` API
   - 更新 `currentBug` 响应式数据
   - 自动渲染详情面板

3. **操作Bug**
   - 通过对话框收集用户输入
   - 调用相应的API (create/update/delete/resolve)
   - 刷新目录树
   - 更新当前Bug详情

### 样式特点

- Flexbox布局实现三栏结构
- CSS变量统一颜色管理
- 渐变色使用linear-gradient
- 自定义滚动条样式
- 响应式媒体查询

## 📝 使用建议

### 最佳实践

1. **合理组织目录结构**
   - 按模块或功能分类
   - 不要嵌套太深 (建议2-3层)
   - 使用清晰的命名

2. **填写完整的Bug信息**
   - 标题简洁明了
   - 描述详细准确
   - 复现步骤清晰
   - 预期/实际结果对比明确

3. **及时更新Bug状态**
   - 开始处理时更新为"处理中"
   - 修复后及时标记为"已解决"
   - 填写解决说明便于追溯

4. **合理设置优先级**
   - P0: 系统崩溃、数据丢失等严重问题
   - P1: 核心功能无法使用
   - P2: 一般功能问题
   - P3: 界面优化、小问题
