# 如何访问测试用例管理模块

## 📍 访问路径

### 方式一：通过项目详情页（推荐）

1. **登录系统**
   - 访问：`http://localhost:5173`
   - 使用管理员账号登录

2. **进入项目列表**
   - 点击顶部导航栏的"项目列表"按钮
   - 或在仪表盘点击"项目管理"按钮
   - 或直接访问：`http://localhost:5173/projects`

3. **选择项目**
   - 在项目列表中点击任意项目的"查看"按钮
   - 进入项目详情页

4. **进入测试用例管理**
   - 在项目详情页左侧菜单中
   - 点击"测试用例"菜单项（带有文档图标）
   - 即可进入测试用例管理页面

### 方式二：直接访问URL

如果你知道项目ID，可以直接访问：

```
http://localhost:5173/projects/{项目ID}/test-cases
```

例如：
```
http://localhost:5173/projects/1/test-cases
http://localhost:5173/projects/2/test-cases
```

## 🔍 常见问题

### Q1: 找不到"测试用例"菜单项
**原因**：可能没有进入项目详情页

**解决方案**：
1. 确认你在项目详情页（URL应该是 `/projects/{id}`）
2. 查看左侧菜单，应该有4个菜单项
3. "测试用例"是第三个，带有📄图标

### Q2: 点击"测试用例"没有反应
**原因**：可能是前端路由配置问题

**解决方案**：
1. 检查浏览器控制台是否有错误
2. 确认前端服务正在运行（`npm run dev`）
3. 刷新页面重试

### Q3: 页面显示404
**原因**：路由未正确配置或项目ID不存在

**解决方案**：
1. 确认项目ID是否正确
2. 检查路由配置文件 `client/src/router/index.js`
3. 确认TestCaseManagement组件已正确导入

### Q4: 页面空白或加载失败
**原因**：后端API未启动或数据库未迁移

**解决方案**：
1. 确认后端服务正在运行（`python run.py`）
2. 确认已执行数据库迁移（`python migrate_add_test_case_module.py`）
3. 确认已初始化权限（`python init_test_case_permissions.py`）

## 🚀 快速开始

如果这是第一次使用，建议按以下顺序操作：

1. **创建模块**
   - 点击左侧"+"按钮
   - 输入模块编号：MOD-USER
   - 输入模块名称：用户模块
   - 点击确定

2. **创建测试用例**
   - 点击中间"新建用例"按钮
   - 填写用例信息
   - 选择刚创建的模块
   - 点击确定

3. **查看用例详情**
   - 在中间列表点击刚创建的用例
   - 右侧会显示用例的完整信息
