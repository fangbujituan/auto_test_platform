# 生产环境部署指南

## 概述

本指南详细说明如何在生产环境中部署自动化测试平台（ATP）。生产环境部署与开发环境不同，需要关注安全性、稳定性、可维护性和性能。

## 部署架构

```
生产服务器
├── Docker Compose
│   ├── mysql-db (MySQL 8.0)
│   ├── flask-app (Flask 后端)
│   └── nginx-frontend (Nginx 前端)
├── 数据持久化
│   ├── /opt/vplatform/mysql_data (MySQL 数据)
│   └── /opt/vplatform/auto_test_platform/logs (应用日志)
└── 网络配置
    └── vplatform_net (Docker 内部网络)
```

## 前置条件

### 服务器要求

- **操作系统**：Ubuntu 20.04+ / CentOS 7+ / Debian 10+
- **内存**：最低 4GB，推荐 8GB+
- **磁盘空间**：最少 20GB 可用空间
- **网络**：稳定的网络连接，开放所需端口

### 软件要求

1. **Docker** 20.10+
2. **Docker Compose** 2.0+
3. **Git** 2.20+
4. **Python 3.13+**（可选，用于管理脚本）

## 部署步骤

### 步骤 1：服务器准备

```bash
# 1. 更新系统
sudo apt update && sudo apt upgrade -y

# 2. 安装 Docker
sudo apt install -y docker.io docker-compose-plugin

# 3. 安装 Git
sudo apt install -y git

# 4. 创建部署目录
sudo mkdir -p /opt/vplatform
sudo chown -R $USER:$USER /opt/vplatform
```

### 步骤 2：获取代码

```bash
# 进入部署目录
cd /opt/vplatform

# 克隆项目（使用 SSH 或 HTTPS）
git clone https://github.com/fangbujituan/auto_test_platform.git

# 进入项目目录
cd auto_test_platform
```

### 步骤 3：环境配置

#### 3.1 创建环境变量文件

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑环境变量
vim .env
```

#### 3.2 关键环境变量配置

以下是必须修改的关键变量：

```env
# ============================================================================
# MySQL 数据库配置
# ============================================================================
MYSQL_HOST=mysql-db              # Docker 内部网络，不要修改
MYSQL_PORT=3306                  # 容器内部端口
MYSQL_USER=atp_prod_user         # 生产环境数据库用户
MYSQL_PASSWORD='StrongPassword123!'  # 强密码，包含大小写字母、数字、特殊字符
MYSQL_DATABASE=atp_production    # 生产环境数据库名
MYSQL_ROOT_PASSWORD='RootStrongPassword456!'  # MySQL root 密码

# ============================================================================
# 部署端口配置
# ============================================================================
BACKEND_PORT=12048               # 后端 API 端口
FRONTEND_PORT=1801               # 前端访问端口

# ============================================================================
# Flask 应用配置
# ============================================================================
FLASK_ENV=production             # 生产环境
SECRET_KEY='FlaskSecretKeyForProduction2026!'  # 强密钥，用于会话加密

# ============================================================================
# AI 模块加密密钥（非常重要！）
# ============================================================================
AI_ENCRYPTION_KEY='your-fernent-key-here'  # 必须是有效的 Fernet 密钥

# ============================================================================
# 安全配置建议
# ============================================================================
# 如果使用 HTTPS，配置 SSL 证书路径
# SSL_CERT_PATH=/etc/ssl/certs/your-domain.crt
# SSL_KEY_PATH=/etc/ssl/private/your-domain.key

# CORS 配置，限制允许的域名
# CORS_ORIGINS=https://your-domain.com,https://admin.your-domain.com
```

#### 3.3 生成 AI_ENCRYPTION_KEY

如果还没有 Fernet 密钥，生成一个：

```bash
# 使用 Python 生成
python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"

# 或者安装 cryptography 后生成
pip install cryptography
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

### 步骤 4：构建和启动服务

```bash
# 1. 构建 Docker 镜像
docker compose build

# 2. 启动所有服务（后台运行）
docker compose up -d

# 3. 查看服务状态
docker compose ps

# 4. 查看日志
docker compose logs -f flask-app
```

### 步骤 5：数据库初始化

```bash
# 等待 MySQL 服务健康检查通过
# 查看状态：docker compose ps

# 1. 初始化数据库表结构
docker compose exec flask-app python -m app.init_db

# 2. 初始化权限系统
docker compose exec flask-app python -m app.init_permission

# 3. 验证数据库连接
docker compose exec mysql-db mysql -uroot -p$MYSQL_ROOT_PASSWORD -e "SHOW DATABASES;"
```

### 步骤 6：验证部署

#### 6.1 检查服务健康状态

## 安全加固

### 1. 修改默认密码

首次登录后立即修改默认密码：
1. 登录系统
2. 进入用户设置
3. 修改管理员密码

### 2. 配置防火墙

```bash
# Ubuntu/Debian 使用 ufw
sudo ufw allow 22/tcp          # SSH
sudo ufw allow 1801/tcp        # 前端端口
sudo ufw allow 12048/tcp       # 后端端口（如果直接暴露）
sudo ufw enable

# CentOS/RHEL 使用 firewalld
sudo firewall-cmd --permanent --add-port=1801/tcp
sudo firewall-cmd --permanent --add-port=12048/tcp
sudo firewall-cmd --reload
```

### 3. 使用 HTTPS（强烈推荐）

#### 使用 Nginx 配置 SSL：

```nginx
# 在 client/nginx.conf 中添加 SSL 配置
server {
    listen 443 ssl;
    server_name your-domain.com;
    
    ssl_certificate /etc/ssl/certs/your-domain.crt;
    ssl_certificate_key /etc/ssl/private/your-domain.key;
    
    # ... 其他配置保持不变
}

# 重定向 HTTP 到 HTTPS
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

#### 获取 SSL 证书：

- **Let's Encrypt**（免费）
- 商业 CA 证书
- 自签名证书（仅测试环境）

### 4. 数据库安全

```sql
-- 创建专门的应用程序用户（已通过环境变量配置）
CREATE USER 'atp_prod_user'@'%' IDENTIFIED BY 'StrongPassword123!';
GRANT SELECT, INSERT, UPDATE, DELETE ON atp_production.* TO 'atp_prod_user'@'%';
FLUSH PRIVILEGES;

-- 限制 root 远程访问
ALTER USER 'root'@'%' IDENTIFIED BY 'RootStrongPassword456!';
```

## 性能优化

### 1. 调整 Docker 资源限制

修改 `docker-compose.yml`：

```yaml
services:
  mysql-db:
    # ... 其他配置
    deploy:
      resources:
        limits:
          memory: 2G
          cpus: '1.0'
        reservations:
          memory: 1G
          cpus: '0.5'
  
  flask-app:
    # ... 其他配置  
    deploy:
      resources:
        limits:
          memory: 4G
          cpus: '2.0'
        reservations:
          memory: 2G
          cpus: '1.0'
```

### 2. 调整 Gunicorn 配置

修改 `Dockerfile` 中的启动命令：

```dockerfile
CMD ["gunicorn", "--bind", "0.0.0.0:12048", "--workers", "4", "--threads", "2", "--timeout", "120", "--worker-class", "gevent", "--access-logfile", "-", "--error-logfile", "-", "run:app"]
```

### 3. 数据库优化

```sql
-- 添加索引优化查询性能
CREATE INDEX idx_test_results_project_id ON test_results(project_id);
CREATE INDEX idx_bugs_project_id ON bugs(project_id);
CREATE INDEX idx_requirements_project_id ON requirements(project_id);
CREATE INDEX idx_test_cases_project_id ON test_cases(project_id);

-- 定期清理旧数据（添加定时任务）
DELETE FROM test_results WHERE created_at < DATE_SUB(NOW(), INTERVAL 90 DAY);
DELETE FROM operation_logs WHERE created_at < DATE_SUB(NOW(), INTERVAL 30 DAY);
```

## 监控和维护

### 1. 日志管理

```bash
# 查看实时日志
docker compose logs -f flask-app

# 查看特定时间段的日志
docker compose logs --since="2024-01-01" --until="2024-01-02" flask-app

# 日志文件位置（宿主机）
/opt/vplatform/auto_test_platform/logs/
├── info.log      # 信息日志
├── error.log     # 错误日志
└── access.log    # 访问日志
```

### 2. 备份策略

#### 数据库备份：

```bash
#!/bin/bash
# /opt/vplatform/backup.sh

BACKUP_DIR="/opt/vplatform/backups"
DATE=$(date +%Y%m%d_%H%M%S)

# 备份数据库
docker compose exec -T mysql-db mysqldump -uroot -p$MYSQL_ROOT_PASSWORD atp_production > $BACKUP_DIR/db_backup_$DATE.sql

# 备份日志文件
tar -czf $BACKUP_DIR/logs_backup_$DATE.tar.gz /opt/vplatform/auto_test_platform/logs/

# 清理旧备份（保留最近30天）
find $BACKUP_DIR -name "*.sql" -mtime +30 -delete
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete
```

#### 设置定时备份：

```bash
# 编辑 crontab
crontab -e

# 添加每日凌晨2点备份
0 2 * * * /opt/vplatform/backup.sh
```

### 3. 健康检查

创建健康检查脚本：

```bash
#!/bin/bash
# /opt/vplatform/health_check.sh

# 检查容器状态
if ! docker compose ps | grep -q "healthy"; then
    echo "ERROR: 有容器不健康" | mail -s "ATP 健康检查失败" admin@example.com
    exit 1
fi

# 检查 API 可用性
if ! curl -s http://localhost:12048/api/health | grep -q "healthy"; then
    echo "ERROR: API 不可用" | mail -s "ATP API 检查失败" admin@example.com
    exit 1
fi

echo "ATP 运行正常"
exit 0
```

## 故障排除

### 常见问题及解决方案

#### 问题 1：端口冲突

**症状**：`docker compose up` 失败，提示端口已被占用

**解决方案**：
1. 修改 `.env` 文件中的端口号
2. 或者停止占用端口的进程

```bash
# 查找占用端口的进程
sudo lsof -i :1801
sudo lsof -i :12048

# 停止进程或修改端口
```

#### 问题 2：数据库连接失败

**症状**：Flask 应用启动失败，提示数据库连接错误

**解决方案**：
1. 检查 MySQL 容器是否运行：`docker compose ps mysql-db`
2. 检查环境变量配置：`cat .env | grep MYSQL`
3. 手动测试数据库连接：

```bash
docker compose exec mysql-db mysql -uroot -p$MYSQL_ROOT_PASSWORD -e "SELECT 1;"
```

#### 问题 3：内存不足

**症状**：容器频繁重启或被 OOM Killer 杀死

**解决方案**：
1. 增加服务器内存
2. 调整 Docker 资源限制（见性能优化章节）
3. 优化应用配置

#### 问题 4：磁盘空间不足

**症状**：无法写入日志或数据库操作失败

**解决方案**：
1. 清理旧日志：`find /opt/vplatform/auto_test_platform/logs -name "*.log" -mtime +30 -delete`
2. 扩展磁盘空间
3. 配置日志轮转

### 紧急恢复流程

1. **停止服务**：`docker compose down`
2. **备份当前状态**：`/opt/vplatform/backup.sh`
3. **检查日志**：`docker compose logs --tail=100 flask-app`
4. **修复问题**：根据日志错误信息修复
5. **重启服务**：`docker compose up -d`
6. **验证恢复**：`/opt/vplatform/health_check.sh`

## 升级流程

### 小版本升级（保持数据）

```bash
# 1. 备份当前状态
/opt/vplatform/backup.sh

# 2. 拉取最新代码
cd /opt/vplatform/auto_test_platform
git pull origin main

# 3. 重建镜像
docker compose build

# 4. 重启服务
docker compose down
docker compose up -d

# 5. 验证升级
docker compose ps
/opt/vplatform/health_check.sh
```

4. 生产环境升级
