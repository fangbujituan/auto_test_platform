# 生产部署问题回顾与安全防护指南

## 一、生产部署问题回顾与解决方案

在 2.0 版本（802 端口）部署过程中，我们主要解决了以下三大类障碍：

### 1. 后端程序启动失败（数据库权限问题）

**问题描述：**
- 后端连接数据库报 `Access denied (1045)` 和 `REFERENCES command denied (1142)`

**根本原因：**
- 数据库用户密码错误
- 项目包含外键约束，普通用户权限不足以执行 `db.create_all()`

**解决办法：**
1. 通过 root 账号在 MySQL 中为用户精准授权：
   ```sql
   GRANT ALL PRIVILEGES ON business.* TO 'yandc'@'%';
   ```
2. 针对 MySQL 8.0 权限不匹配（Error 1410）问题：
   - 确认 `host` 字段正确
   - 执行 `FLUSH PRIVILEGES;` 刷新权限表

---

### 2. 端口与进程冲突问题

**问题描述：**
- 启动时报 `Address already in use`，端口 12048 被占用

**根本原因：**
- 后台残留了旧的 Python 进程或 Gunicorn 守护进程

**解决办法：**
```bash
# 方法1：直接杀掉端口进程
fuser -k 12048/tcp

# 方法2：先查 PID 再杀
netstat -nlp | grep :12048
kill -9 <PID>
```

---

### 3. Nginx 转发路径错误（405 报错）

**问题描述：**
- 访问登录接口时返回 `405 Not Allowed`

**根本原因：**
- Nginx 配置的 `location /auto/` 与前端实际请求的 `/api/` 不符
- POST 请求被当作静态资源请求处理

**解决办法：**
```nginx
# 修改前（错误）
location /auto/ {
    proxy_pass http://127.0.0.1:12048/;
}

# 修改后（正确）
location /api/ {
    proxy_pass http://127.0.0.1:12048;
}
```

**关键点：**
- `proxy_pass` 后面**不加末尾斜杠**，确保完整透传路径
- 确保 location 匹配前端实际请求路径

## 二、公网安全风险总结

即便你不担心测试数据丢失，公网环境下的主要风险在于虚拟机可能被作为跳板影响宿主机和局域网。

| 攻击类型 | 描述 | 影响 |
|---------|------|------|
| **自动化扫描脚本** | 扫描全网端口，寻找开放服务 | 暴露服务指纹，发现漏洞 |
| **路径/暴力破解** | 尝试 `/admin`, `/.env` 或循环尝试 API 登录密码 | 账号被窃取，控制后台 |
| **MITM 中间人攻击** | HTTP 明文传输，截获账号密码 | 敏感信息泄露 |
| **横向移动** | 黑客攻陷虚拟机后，扫描局域网内的手机、NAS、个人电脑 | 危及内网其他设备安全 |
| **宿主机逃逸** | 利用虚拟化软件漏洞，通过虚拟机控制宿主机 | 控制整台电脑系统 |

## 三、安全防护工具箱与策略

### 1. 核心防护工具

#### Fail2Ban（防爆破神器）
- **作用**：监控日志文件，一旦发现某个 IP 频繁尝试登录失败（如 1 分钟内 5 次），自动通过防火墙封禁该 IP
- **防护对象**：SSH 登录、Web 接口登录

#### Tailscale / ZeroTier（内网穿透）
- **作用**：如果不是必须对外公开，可以使用这类工具组建虚拟私有网
- **优势**：关闭公网端口映射，只有你信任的设备能访问 802 端口

#### Certbot（HTTPS 加密）
- **作用**：自动申请 Let's Encrypt 免费 SSL 证书
- **优势**：解决明文传输问题，防止中间人嗅探

---

### 2. 攻击检测工具（知己知彼）

你可能会在日志中看到这些工具的痕迹：

| 工具 | 用途 |
|-----|------|
| **ZMap / Masscan** | 黑客常用的全网端口快速扫描器 |
| **Dirsearch / FFuf** | 用于爆破隐藏路径（如扫描 `/api`, `/backup` 等） |

---

### 3. 架构级加固策略（防止影响宿主机）

| 策略 | 说明 |
|-----|------|
| **最小化权限** | 严禁使用 root 运行 Python 项目。创建一个受限用户（如 `webuser`）专门跑程序 |
| **Docker 容器化** | 将应用封在容器内，即便程序被破，黑客也很难触碰到容器外的虚拟机文件系统 |
| **隔离虚拟机组件** | 关闭虚拟机设置中的"剪贴板共享"、"拖放文件"和"共享文件夹" |
| **NAT 模式** | 将虚拟机网卡设为 NAT 而非桥接，减少其在物理局域网中的暴露面 |

---

## 四、最终操作建议

### 立即执行（高优先级）
1. **改运行用户**：别用 root 跑项目
   ```bash
   # 创建受限用户
   sudo adduser webuser
   sudo chown -R webuser:webuser /path/to/project
   # 使用 webuser 启动应用
   sudo -u webuser python run.py
   ```

2. **安装 Fail2Ban**
   ```bash
   # macOS
   brew install fail2ban
   
   # Ubuntu/Debian
   sudo apt install fail2ban
   
   # 配置 Web 接口防护（/etc/fail2ban/jail.local）
   [http-auth]
   enabled = true
   port = http,https
   filter = http-auth
   logpath = /var/log/nginx/error.log
   maxretry = 5
   bantime = 3600
   findtime = 60
   ```

3. **配置 Nginx 限流**
   ```nginx
   # 限制请求速率（每秒10个请求）
   limit_req_zone $binary_remote_addr zone=api_limit:10m rate=10r/s;
   
   location /api/ {
       limit_req zone=api_limit burst=20 nodelay;
       proxy_pass http://127.0.0.1:12048;
   }
   ```

### 后续优化（中优先级）
- 配置 HTTPS（使用 Certbot）
- 迁移到 Docker 容器化部署
- 配置 Tailscale/ZeroTier 内网访问
- 关闭虚拟机共享功能，改用 NAT 网络模式
