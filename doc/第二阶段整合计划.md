## 🔧 整合策略

### 原则
1. **先基础后复杂** - 先整合工具模块，再整合中间件，最后整合 Playwright
2. **保持兼容** - 不影响主项目现有功能
3. **模块化移植** - 每个模块独立移植，独立测试
4. **依赖管理** - 先处理依赖模块，再处理被依赖模块

### 依赖关系
```
dom_utils.py
    ↓
selector_scores.py
    ↓
semantic_selector.py
    ↓
dom_cleaner.py, code_collector.py
```
