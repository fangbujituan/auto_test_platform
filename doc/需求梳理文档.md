# 自动化测试平台 (ATP) 需求梳理文档

---

## 一、项目概述

**项目名称**：自动化测试平台 (Automation Test Platform, ATP)

**核心定位**：基于 Flask + Vue 的 AI 驱动自动化测试平台，支持 API 管理、测试用例、Bug 跟踪、用例执行和结果分析。

**愿景**：用户在页面的对话框中描述测试意图，平台即可自动生成测试用例与可执行脚本，覆盖三类测试：
- UI 测试
- API 测试  
- 性能测试

---

## 二、系统架构

### 2.1 技术栈

**后端技术栈**：
- Python 3.10+
- Flask 3.x (Web 框架)
- Flask-SQLAlchemy (ORM)
- Flask-Smorest (OpenAPI 文档生成)
- Marshmallow (请求/响应 Schema)
- MySQL (数据库)
- PyMySQL (数据库驱动)
- Requests (HTTP 客户端)
- LangGraph (Agent 编排)
- LangChain (LLM 集成)

**前端技术栈**：
- Vue 3
- Vite
- Element Plus (UI 组件库)
- Axios (HTTP 客户端)
- Vue Router (路由管理)

**AI 能力技术栈**：
- 多模型网关支持 (AIOP Gateway/Kiro Gateway/AIClient2API)
- Playwright (浏览器自动化)
- MCP 协议 (Model Context Protocol)
- FastAPI/Uvicorn (Agent 服务)

## 三、功能需求

### 3.1 用户认证与权限管理

**3.1.1 用户认证**
- 用户登录/登出
- 密码加密存储 (Werkzeug)
- 用户激活/禁用

**3.1.2 权限模型 (RBAC)**
- 平台管理员 (Admin)：所有权限
- 项目负责人 (Owner)：项目内所有权限
- 项目成员 (Member)：读写权限，不能删除项目和管理成员
- 只读用户 (Viewer)：仅查看权限

**3.1.3 项目成员管理**
- 添加/移除项目成员
- 成员角色分配
- 成员列表展示

### 3.2 项目管理

**3.2.1 项目 CRUD**
- 创建项目
- 编辑项目信息
- 删除项目
- 项目列表展示
- 项目状态管理 (启用/禁用)

**3.2.2 项目配置**
- 项目描述
- 项目级环境变量管理
- AI 提供商配置

### 3.3 API 管理

**3.3.1 API 接口管理**
- API 增删改查
- API 信息：名称、描述、方法、路径、基础 URL
- 请求配置：Headers、Params、Body、Body 类型
- 响应示例
- 模块/服务分类
- API 状态管理

**3.3.2 API 目录管理**
- 树形目录结构
- 目录增删改查
- API 归类到目录

**3.3.3 API 导入**
- 支持 cURL 命令导入
- 支持 Swagger/OpenAPI JSON 导入
- 导入预览与确认

**3.3.4 API 在线测试**
- 在线发送 API 请求
- 查看响应结果
- 保存测试结果

### 3.4 测试用例管理

**3.4.1 API 测试用例**
- 用例名称、描述
- 请求配置 (方法、URL、Headers、Params、Body)
- 断言配置 (预期状态码、预期响应)
- 项目关联
- 优先级管理 (高/中/低)

**3.4.2 测试用例管理 (功能用例)**
- 用例编号自动生成
- 用例标题、描述
- 前置条件、测试步骤、预期结果
- 模块/目录分类
- 用例优先级 (P0/P1/P2/P3)
- 用例类型 (功能/性能/安全等)
- 用例状态 (草稿/已评审/已废弃)

**3.4.3 用例-API 绑定**
- 支持将功能用例绑定到多个 API
- 支持执行顺序配置
- 支持备注信息

### 3.5 Bug 管理

**3.5.1 Bug 信息管理**
- Bug 标题、描述
- 项目关联
- 状态流转 (open/in_progress/resolved/closed/reopened)
- 优先级 (low/medium/high/critical)
- 严重程度 (trivial/minor/normal/major/critical)
- Bug 分类
- 模块/目录关联
- 标签管理

**3.5.2 人员与环境**
- 报告人
- 指派人
- 测试环境
- 发现版本

**3.5.3 复现信息**

- 预期结果
